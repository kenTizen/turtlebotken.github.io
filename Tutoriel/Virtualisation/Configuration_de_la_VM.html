Configuration de la VM

Les instructions suivantes vont permettre à l'utilisateur de configurer sa machine virtuelle d'un point de vue réseau

NAT ou Bridge

Un tour dans les propriétés réseau d'une machine virtuelle permet de visualiser les différents modes d'accès réseau disponibles : NAT, accès par pont, réseau interne, réseau privé hôte, Generic Driver, réseau NAT, Cloud Network, aucune connexion. Avec autant de choix, il y a de quoi répondre à de nombreux besoins, mais aussi de perdre de nombreux utilisateurs.

Le mode NAT permet à votre VM d'acceder à Internet, via la même connexion que l'hôte physique, sans pour autant obtenir une adresse IP sur votre réseau local. Effectivement, c'est le serveur DHCP de VirtualBox ou VMWare qui va attribuer une adresse IP à votre machine virtuelle, et non le serveur DHCP de votre réseau local. Lorsque ce mode est utilisé, VirtualBox crée un réseau local virtuel et isolé du reste de vos machines physiques.

En sélectionnant le mode "Accès par pont" que l'on peut appeler le mode "Bridge", la machine virtuelle aura un accès à votre réseau, au même titre que votre ordinateur physique. De ce fait, si elle est configurée en DHCP, elle va solliciter le serveur DHCP de votre réseau local pour obtenir une adresse IP et accéder au réseau local. Avec un accès par pont, la VM pourra contacter les autres machines connectées au réseau et elle pourra être contactée par les autres machines de ce réseau, contrairement au mode NAT où ce n'était pas possible (à moins de créer une ou plusieurs règles de redirection de ports).

Choix du mode de connexion

Nous allons choisir le mode "Accès par pont" ou "Bridge" pour notre machine virtuelle afin d'être dans le même reseaux local que l'hôte et le Robot (à condition que celui-ci soit dans le même reseau que l'hôte)
